## 撤销修改

### 撤销上一次提交

```bash
git commit --amend
```

适用于上次提交遗漏了文件的场景，因为他会将暂存区的所有文件提交掉，并用第二次提交替换掉第一次。

### 已将文件添加到暂存区

如果已经通过 `git add *` 将不想暂存的文件暂存了，可以通过

```bash
git reset HEAD <file>
```

来取消暂存。

### 撤销本地修改

```bash
git checkout -- <file>
```

该命令会使用最近一次提交的文件来覆盖本地的文件，会导致本地文件的任何修改丢失。
